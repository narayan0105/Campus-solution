# Smart Issue Reporting Platform – MVP

## 1. Problem Statement
In campuses and local communities, infrastructure issues such as broken lights, water leakage, and safety hazards are often reported informally or ignored. This leads to slow resolution, lack of accountability, and frustration among students and residents.

## 2. MVP Goal
The goal of this MVP is to provide a simple, working web platform that allows users to report issues and view them transparently using Google technologies.

## 3. Core Features Implemented
- Users can submit an issue with a title and description
- Issues are stored centrally using Firebase Firestore
- Reported issues are displayed in a list view
- Google Maps shows issue locations
- Clean and professional user interface

## 4. Google Technologies Used
- Firebase Firestore – data storage
- Firebase Hosting – web hosting
- Google Maps API – location visualization

## 5. Real-World Impact
- Faster reporting of issues
- Improved transparency
- Centralized issue tracking for institutions

## 6. Future Improvements
- AI-based issue classification using Google Cloud Vision API
- Admin dashboard for managing issues
- Authentication and role-based access control
